description: 快速提交代码并推送到 GitHub

将当前更改提交并推送到 GitHub 仓库。

**输入**: `/push` 后面可以跟可选的提交消息，例如 `/push 修复登录bug`

**步骤**

1. **查看当前状态**
   ```bash
   git status
   ```

2. **查看更改内容**
   ```bash
   git diff
   ```
   同时查看最近的提交记录以了解提交风格：
   ```bash
   git log --oneline -5
   ```

3. **确定提交消息**

   **如果用户提供了提交消息**（例如 `/push 修复登录bug`）：
   - 使用用户提供的消息作为提交主题

   **如果用户没有提供提交消息**：
   - 分析 git diff 的内容，自动生成一个简洁的中文提交消息
   - 提交消息应该描述**做了什么**，而不是**怎么做**
   - 使用祈使句，例如：
     - "添加用户认证功能"
     - "修复页面样式问题"
     - "更新配置文件"

4. **添加文件到暂存区**

   ```bash
   git add .
   ```

   **注意**：如果有敏感文件（如 .env、密钥文件等），询问用户是否要排除这些文件。

5. **创建提交**

   使用以下格式创建提交：
   ```bash
   git commit -m "$(cat <<'EOF'
<提交消息>

Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>
EOF
)"
   ```

6. **推送到远程仓库**

   ```bash
   git push
   ```

7. **验证推送结果**

   ```bash
   git status
   ```

**输出**

完成后的总结：
- ✅ 提交的哈希值
- ✅ 提交消息
- ✅ 推送的分支信息
- ✅ 修改的文件数量和行数
- ✅ GitHub 仓库链接

**注意事项**

- 如果没有更改需要提交，告知用户工作目录是干净的
- 如果推送失败（例如认证问题、网络问题），提供错误信息和建议的解决方案
- 如果有未跟踪的文件，会自动添加，但敏感文件需要用户确认
- 提交消息使用中文，保持简洁明了（通常一行就够了）
